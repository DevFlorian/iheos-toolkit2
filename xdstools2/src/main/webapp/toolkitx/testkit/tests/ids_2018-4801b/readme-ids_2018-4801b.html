RAD-68: PnR KOS for Multi Image Study

Tests the ability of the Image Document Source actor (SUT) to send a Provide and Register Imaging Document Set (RAD-68) transaction to a Repository/Registry actor (Simulator). The Imaging Document Source is required to submit a DICOM KOS object that references an imaging study with multiple images in different series.. As part of this test, you will need to map the Patient Identifier value found in the DICOM image to the DocumentEntry.patientId value in the XDS metadata.

Mappings

C3N-00953 IDS_2018-4801b

Instructions

This test follows the same pattern as ids_2018-4801a. Follow the instructions from ids_2018-4801a with these differences:
  • Data set is C3N-00953
  • Patient identifier mapping is shown in the table above.

Notes

The KOS object for this test will reference multiple images in multiple series.
Some of the metadata elements that are based on the original image content (e.g., Accession Number) will differ from the values in ids_2018-4801a.
